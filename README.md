## 🏗️ Главный проект: Автоматизированный мониторинг системы (`00_Automated_DevOps_Infrastructure`)

Центральный проект портфолио — отказоустойчивый комплекс для сбора метрик хоста и автоматического деплоя. 

### Архитектура комплекса:
* **Агент мониторинга (Python):** Фоновый скрипт, использующий библиотеки `psutil` и `shutil`. Собирает данные о RAM (%) и дисковом пространстве виртуалки Ubuntu. Оснащен системой алертинга в Telegram при критических нагрузках (>90%).
* **База данных (PostgreSQL):** Контейнеризированная СУБД для хранения логов мониторинга. Данные защищены от потери при перезапуске через Docker Volumes.
* **Веб-витрина (FastAPI):** Backend-интерфейс, который поднимает метрики из базы и выводит их в веб-интерфейс.

### Автоматизация и CI/CD:
Вся инфраструктура полностью упакована в **Docker Compose** с изолированными внутренними сетями. Настроен автоматический пайплайн деплоя через **GitLab CI**: при пуше в ветку `main`, робот валидирует конфигурацию докера (`docker compose config`), подключается к удаленному серверу по SSH, делает `git pull` и перезапускает проект.

## 🛠️ Структура портфолио по модулям

* **`00_Automated_DevOps_Infrastructure`** — Главный инфраструктурный DevOps-проект (Docker Compose, Python, PostgreSQL, FastAPI, GitLab CI).
* **`01_Linux_Network`** — Практические задачи по настройке сетевых интерфейсов, маршрутизации, DNS, портов и базовой безопасности Linux.
* **`02_Linux_Monitoring_Base`** — Скрипты автоматизации на Bash, работа с логами операционной системы и системными утилитами мониторинга.
* **`03_SQL_Database_Design`** — Проектирование реляционных баз данных, создание таблиц, управление пользователями и правами доступа (`chown`/`chmod`/`GRANT`).
* **`04_SQL_Data_Retrieval` & `05_SQL_Advanced_Queries`** — Работа с выборками данных, написание сложных и оптимизированных SQL-запросов.
* **`06_C_Language_Basics` & `07_Team_Project_C_Ping_Pong`** — Фундаментальный бэкграунд низкоуровневого программирования. Изучение работы с памятью, указателями и процессами ОС на языке Си (базовый этап закалки инженера).
